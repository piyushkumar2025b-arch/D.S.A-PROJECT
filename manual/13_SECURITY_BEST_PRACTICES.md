# 🛡️ 13. Security Best Practices

Since SAAP connects to deep workspace APIs, security is paramount.

## Protecting Your API Keys
*   **Environment Variables**: Never hard-code keys in `app.py`. Always use `.streamlit/secrets.toml`.
*   **.gitignore**: Ensure your secrets file is ignored by Git so you don't accidentally push your Slack or GitHub tokens to a public repo.
*   **Least Privilege**: When creating API keys (like in JIRA or HubSpot), only give the "Read" or "Write" permissions specifically needed. Do not use "Super Admin" tokens.

## User Access Control
*   **Section 5 Protection**: Only share Section 5 credentials with trusted administrators.
*   **Audit Logs**: Regularly review the logs in **Section 9** to see who is calling which agent and if any unusual patterns of data extraction are occurring.

## Data Processing
SAAP does not store your private messages or emails in its permanent database—it only stores the "Summary Report" generated by the AI for history tracking. Your raw private data remains in the source platform (Slack/Gmail).
