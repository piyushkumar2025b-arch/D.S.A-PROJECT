# 🤖 SAAP — Smart Autonomous Agent Platform
### 🔬 Research Prototype / Proof-of-Concept 
Powered by Groq (FREE, Llama 3.3 70B)

> **⚠️ RESEARCH PROTOTYPE NOTICE:** This platform is completely experimental and acts as an **AI simulation sandbox**. It does **NOT** connect to actual live accounts (Gmail, Slack, Jira, etc.). Instead, it uses Large Language Models natively optimized for structured JSON generation to realistically *simulate* the outputs, decisions, and workflows of 12 distinct autonomous agents interacting with mock environments.

---

## 🏗️ System Architecture

The architecture is designed to be lightweight, observable, and modular for rapid research and testing:

1. **Frontend (Streamlit):** Defines the UI pages (Dashboard, Pipeline Builder, Playground) and provides a sandbox interface to view complex JSON and run test environments.
2. **State Management (SQLite):** Acts as the central "memory" for the system. It tracks:
   - Available Agent configurations and prompts
   - A persistent log of `Task History` (payloads in, JSON out, runtime metrics)
   - Created `Pipelines` (which agents run in what sequence)
3. **Brain / Inference Engine (Groq LPU API):** 
   - All agents are powered purely by a single LLM connection: **Llama 3.3 70B Versatile** operating over Groq for ultra-low latency. 
   - We inject strict system prompts and instructions depending on the chosen agent, enforcing that the model outputs strictly formatted, production-ready simulated JSON.

---

## 🤖 Agent Usage & Operation Flow

An **Agent** in this system is essentially a highly-specialized prompt persona wrapped around the LLM, designed to ingest a structured user mandate, mock the external tool interaction, and return a perfectly structured response payload.

### Step-by-Step Flow:
1. **Selection:** You choose an agent persona (e.g., `gmail-summary`).
2. **Payload Construction:** You provide a JSON input dictionary dictating what the agent is supposed to do.
3. **Execution:** The system builds a unified prompt combining the generic agent persona with your specific payload, and fires it to Groq. 
4. **Schema Output:** The Groq Llama 3.3 model generates a hyper-realistic JSON response mimicking the exact expected return structure of the requested tool (like simulating an array of Jira tickets with priority tags).

## 📥 Agent Inputs (How to Command the Agents)

Every interaction happens via **JSON Payloads**. Because this is a simulation, you do not query databases directly; instead, you pass simple instructions simulating what an API call to that agent *might* look like in production.

Here are examples of what you input to the agents (these can be modified directly in the **🚀 Run Agent** or **🧪 Playground** tabs):

**1. `gmail-summary`**
```json
{
  "action": "summarize_unread",
  "max_emails": 25,
  "label": "INBOX",
  "include_drafts": false
}
```
*Expected Output:* A simulated JSON digest of top senders, high-priority issues, and categorized themes.

**2. `github-agent`**
```json
{
  "action": "pr_digest",
  "repo": "org/backend",
  "days": 7,
  "include_reviews": true
}
```
*Expected Output:* Valid JSON mapping simulated Pull Requests, status, code reviews, additions/deletions, and CI states.

**3. `sheets-agent`**
```json
{
  "action": "create_report",
  "title": "Weekly Task Report",
  "include_charts": true,
  "sheet": "Sheet1"
}
```
*Expected Output:* Mocked column/row array data structure and report highlights.

**4. `slack-agent`**
```json
{
  "action": "summarize_channel",
  "channel": "#engineering",
  "hours": 24,
  "include_threads": true
}
```
*Expected Output:* Extracted key decisions, active attendees, and action items generated by the LLM simulation.

> Notice that in the **🔗 Pipeline Builder**, you can chain these inputs. The structured output of `web-scraper` can automatically be injected into the next LLM step, powering the `notion-agent` to create a compiled page of the scraped results!

## 🔑 Getting Started (No external keys needed!)

You only need **ONE** API key to power this entire simulation environment: The Groq API Key.

1. Go to **[console.groq.com](https://console.groq.com)**
2. Sign up and click **API Keys** → **Create API Key**
3. Copy your key that starts with `gsk_…`

Because we use simulation, you do **not** need API keys for Google, Slack, Hubspot, or Jira.

## 💻 Run Locally

```bash
# Clone the repository
git clone <your-repo-link>
cd <repo-folder>

# Install dependencies 
pip install -r requirements.txt

# Run the research platform
streamlit run app.py
```
Open **http://localhost:8501** in your browser. 

*Note: You can paste your Groq API Key into the app's sidebar, or securely link it by creating a `.streamlit/secrets.toml` with `GROQ_API_KEY="gsk_..."`.*

## 🎯 Research Objectives & Focus Areas

1. **Structured Outputs & Formatting:** Testing the capability of Llama-3.3-70B to return perfectly formatted JSON documents without schema hallucinations.
2. **Chain-of-Thought Pipeline Processing:** Evaluating how effectively one agent's raw JSON output acts as the implicit "context" for a subsequent agent within an automated pipeline chain.
3. **Prompt Specificity:** Analyzing how well a single LLM can context-switch its schema generation to accurately reflect 12 disparate enterprise APIs.
4. **Latency:** Utilizing Groq's LPU architecture to benchmark the processing speed.
